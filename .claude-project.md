# MLR Analysis Project - Claude Code Context

## Project Overview
This is a comprehensive Medical Loss Ratio (MLR) analysis dashboard for Clearline International Limited, a health insurance company (HMO).

## Key Files
- **MLR.py** - Main Streamlit dashboard for MLR analysis
- **dlt_sources.py** - Data loading pipeline from MediCloud and EACOUNT databases
- **auto_update_database.py** - Database schema management and updates
- **ai_driven_data.duckdb** - Main DuckDB database (all tables in "AI DRIVEN DATA" schema)

## Database Schema
All tables are in the `"AI DRIVEN DATA"` schema. Key tables:
- `CLAIMS DATA` - Medical claims (use `nhisgroupid` to join with groups)
- `PA DATA` - Prior authorizations
- `GROUPS` - Client/company information
- `GROUP_PLANS` - Plans per group (columns: `countofindividual`, `countoffamily`, `individualprice`, `familyprice`)
- `MEMBER_PLANS` - Member plan assignments
- `PLANS` - Plan definitions
- `DIAGNOSIS` - Diagnosis codes and descriptions
- `PROCEDURE_DATA` - Procedure codes and descriptions
- `PROVIDERS` - Hospital/provider information

## Important Notes
- Always use schema prefix: `"AI DRIVEN DATA"."TABLE_NAME"`
- Claims use `nhisgroupid` (not `groupid`) to join with GROUPS
- Claims procedure column is `code` (not `procedurecode`)
- GROUP_PLANS uses `countofindividual`/`countoffamily` (not `individualcount`/`familycount`)
